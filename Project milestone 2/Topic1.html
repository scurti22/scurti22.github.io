Topic 1

The history of HTTP

This page will go over the history of HTTP and the versions that lead up to what we have today.

HTTP has gone through several versions since its creation in 1989 these versions include.

  • The One-Line Protocol
  • HTTP 1.0
  • HTTP 1.1
  • HTTP 2
  • HTTP 3

HTTP was created along with three other building blocks. HTML was created to represent hypertext documents, WorldWideWeb to display and edit, and httpd which gives access to the document. HTTP’s place was to exchange documents. HTTP was termed the one-line protocol.

The One Line Protocol

The one-line protocol consisted of a single line, for example “get / my-page.html”. Using this protocol had its downsides as it could only transmit HTML files. With no status or error codes, it created HTML files with a description of the issue.

HTTP1.0

The One-Line Protocol was short lived, it was quickly updated into HTTP/1.0 to include more versatility. Allowing other types of documents to be transmitted using a content type header. Headers were initially introduced for requests and responses which created more flexibility and extensibility.

The Standardized Protocol

Standardized protocol was the 3rd version of HTTP and was published in 1997. It added several different elements to improve the protocol. Including reusable connections, pipelining, which allowed multiple requests to be sent at the same time, lowering latency. Chunked responses, cache control mechanisms, content negotiation, and a host header that allows different domains from the same IP addresses, allowing server collocation. The standardized protocol is when HTTP began to be shaped into how we know it today. Being revised two times. In 2022 HTTP was split into 5 versions. Also achieving the status of internet standard.
